# Архитектура системы AI Agents

Простым языком: как всё устроено.

---

## Что это вообще такое?

Это система из нескольких "агентов" — программ, которые работают автономно (сами по себе).
Каждый агент делает одну задачу и передаёт результат следующему агенту.

Представь конвейер на заводе:
- Робот #1 находит детали
- Робот #2 проверяет качество
- Робот #3 собирает продукт

У нас так же, только с бизнес-нишами:
- **Agent #1** (Niche Researcher) — находит бизнес-ниши
- **Agent #2** (будущий) — глубоко анализирует найденные ниши
- **Agent #3+** (будущие) — запускают бизнес

---

## Как работает Agent #1?

```
Шаг 1: Ищет в Google свежие тренды
        (использует SerpApi — это как программный доступ к Google)
             ↓
Шаг 2: Отправляет тренды в GPT-4o (мозг агента)
        GPT-4o придумывает 50 бизнес-идей на основе трендов
             ↓
Шаг 3: Проверяет — вдруг эти идеи уже были раньше?
        (сверяет с базой данных Supabase)
             ↓
Шаг 4: Фильтрует — отсеивает плохие ниши
        (12 жёстких правил: маржа, капитал, рынок и т.д.)
             ↓
Шаг 5: Оценивает — считает балл каждой ниши
        (формула учитывает маржу, рост, автоматизацию)
             ↓
Шаг 6: Сохраняет ТОП-10 в базу данных
```

## Где что лежит?

```
ai-agents/
│
├── docs/           — ты сейчас здесь. Документация
├── sql/            — SQL-скрипты для создания таблиц в базе
│
└── src/            — исходный код (то что выполняется)
    │
    ├── shared/     — общие модули (используются всеми агентами)
    │   ├── config.ts    — загружает секретные ключи из .env файла
    │   ├── openai.ts    — подключение к GPT-4o
    │   ├── supabase.ts  — подключение к базе данных
    │   ├── logger.ts    — записывает логи (что происходит и когда)
    │   └── types.ts     — описание структур данных (как выглядит "ниша")
    │
    └── agents/
        └── niche-researcher/  — Agent #1
            ├── index.ts       — ГЛАВНЫЙ ФАЙЛ. Запускает всё по порядку
            ├── schemas.ts     — описывает формат данных для GPT-4o
            ├── queries.ts     — список поисковых запросов для Google
            ├── search.ts      — ищет тренды в Google
            ├── generate.ts    — просит GPT-4o придумать ниши
            ├── deduplicate.ts — убирает повторы
            ├── filter.ts      — применяет 12 фильтров
            ├── score.ts       — считает итоговый балл
            └── store.ts       — сохраняет в базу данных
```

## Технологии (простым языком)

| Технология | Что это | Зачем нам |
|-----------|---------|----------|
| **TypeScript** | Язык программирования (улучшенный JavaScript) | Пишем код агентов |
| **Node.js** | Среда для запуска TypeScript/JavaScript | Запускает наш код |
| **GPT-4o** | Искусственный интеллект от OpenAI | "Мозг" агента — анализирует и генерирует |
| **Supabase** | Облачная база данных (PostgreSQL) | Хранит все ниши, циклы, результаты |
| **SerpApi** | Программный доступ к Google Search | Агент ищет тренды без браузера |
| **Zod** | Библиотека проверки данных | Гарантирует что GPT-4o вернёт правильный формат |

## Как агенты общаются между собой?

Через **базу данных** (Supabase). Это как общий склад:
- Agent #1 кладёт ниши на склад (таблица `niches`)
- Agent #2 берёт ниши со склада, анализирует, обновляет статус
- Если Agent #2 отклоняет нишу — записывает причину (таблица `rejections`)
- Agent #1 читает отклонения и больше не предлагает похожие ниши

Никакой прямой связи между агентами нет. Всё через базу.
