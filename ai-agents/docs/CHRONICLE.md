## 27 февраля 2026

### Решение: чистый код вместо n8n
- Изначально планировали строить агентов в n8n (визуальный конструктор)
- Проблема: для 10+ агентов n8n становится ограничивающим — сложная логика, нет тестов, нет git
- Решение: пишем на TypeScript (JavaScript с типами). Полный контроль над кодом

### Начало работы над Agent #1: Niche Researcher
- Что делает: ищет 50 бизнес-ниш в интернете, фильтрует, оценивает, сохраняет лучшие 10
- Алгоритм: Google → GPT-4o (анализ) → фильтры → оценка → Supabase (база)

### Код написан, прошёл type-check
- 5 shared модулей (config, logger, openai, supabase, types)
- 8 модулей агента (schemas, queries, search, generate, dedup, filter, score, store + index)
- SQL-схема для 3 таблиц (cycles, niches, rejections)
- Документация: architecture, how-to-run, glossary

### Ревью кода — найдены и исправлены 4 бага
1. **Дубликаты между batch-ами GPT-4o** — два параллельных вызова могли вернуть одинаковые ниши. Добавлена дедупликация после слияния batch-ей
2. **Batch-и не различались по содержанию** — оба получали одинаковый промпт. Теперь batch 1 фокусируется на SaaS/tools, batch 2 на инфопродуктах/маркетплейсах
3. **`max_tokens` deprecated** — заменён на `max_completion_tokens` для GPT-4o
4. **Двойная сортировка** — и index.ts и store.ts сортировали top-10. Убрана лишняя сортировка в store.ts
5. **SerpApi запросы были последовательными** — переделаны на параллельные (Promise.allSettled) для скорости

### Второе ревью — ещё 4 бага (runtime edge cases)
1. **GPT-4o refusal** — если модель откажется генерировать (content policy), ошибка не показывала причину. Добавлена проверка `message.refusal`
2. **Нет unique constraint в БД** — дедупликация была только в коде, не в базе. Добавлен `UNIQUE INDEX` на `lower(niche_name)` — база не пропустит дубликат даже при сбое в коде
3. **Пустые сниппеты** — если SerpApi вернёт 0 сниппетов (запросы успешны но пустые), GPT-4o генерировал бы из памяти без реальных трендов. Добавлена проверка `totalSnippets === 0`
4. **Невалидный LOG_LEVEL** — если в .env написать `LOG_LEVEL=verbose`, logger молча ломался. Добавлена валидация: неизвестный уровень → fallback на "info"
