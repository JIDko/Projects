# Словарь терминов

Все технические слова которые встречаются в проекте, простым языком.

---

| Термин | Что это |
|--------|---------|
| **Agent (агент)** | Программа которая работает сама по себе, выполняя определённую задачу |
| **API** | "Дверь" через которую одна программа общается с другой. Как окошко в банке |
| **API Key** | Пароль для доступа через API. У каждого сервиса свой ключ |
| **CLI** | Command Line Interface — управление программой через текстовые команды в терминале |
| **Cycle (цикл)** | Один полный запуск агента от начала до конца |
| **Dedup (дедупликация)** | Удаление повторов. Если ниша уже была — не предлагать снова |
| **dotenv** | Библиотека которая загружает секретные ключи из файла .env |
| **.env файл** | Файл с секретными ключами. НИКОГДА не загружается в git |
| **GPT-4o** | Модель ИИ от OpenAI. "Мозг" наших агентов |
| **LLM** | Large Language Model — большая языковая модель. GPT-4o это LLM |
| **Node.js** | Программа которая умеет запускать JavaScript/TypeScript код |
| **npm** | Менеджер пакетов для Node.js. Устанавливает библиотеки |
| **Pipeline** | Конвейер — данные проходят через несколько шагов по порядку |
| **PostgreSQL** | База данных. Supabase работает на PostgreSQL |
| **Schema (схема)** | Описание структуры данных. Как чертёж для данных |
| **SerpApi** | Сервис который даёт программный доступ к поиску Google |
| **Structured Output** | Режим GPT-4o когда он ОБЯЗАН вернуть данные в точном формате (JSON) |
| **Supabase** | Облачная база данных + API. Как Firebase но на PostgreSQL |
| **Token (токен)** | Единица текста для LLM. Примерно 1 токен ≈ 4 символа |
| **tsx** | Утилита для запуска TypeScript файлов напрямую, без компиляции |
| **TypeScript** | Язык программирования. JavaScript с проверкой типов данных |
| **UUID** | Уникальный идентификатор. Как серийный номер для каждой записи |
| **Webhook** | URL на который можно отправить HTTP-запрос чтобы что-то запустить |
| **Zod** | Библиотека для описания и проверки формата данных в TypeScript |
